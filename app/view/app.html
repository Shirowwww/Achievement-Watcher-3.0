        Emulator setup (GBE Fork)

        Defaults used by the right-click Apply emulator fix action. Choose how Steam is bypassed and how much game data is fetched.

        Core configuration
        • Automatically fix newly detected games
          Runs the full Apply emulator fix chain once for a newly detected, unconfigured emulated game.
        • Emulator mode
          AW swaps the game's steam_api(64).dll with GBE Fork's (standalone).
        • steam_settings data
          Simple uses Achievement Watcher's own fetch (anonymous). Advanced runs generate_emu_config for depot/language coverage.
        • Data source (Advanced)
          Steam login lets generate_emu_config pull complete DLC/depot data. Use a THROWAWAY account only — never your main.
        Steam login (throwaway account only)
        Throwaway account

        Used only when "Data source" is set to Steam login. The password is stored encrypted on this PC and sent only to Steam by generate_emu_config. Steam Guard 2FA is prompted when needed. Leave empty to be asked each time.

        Advanced options
        • Auto-unpack SteamStub DRM
          Runs Steamless on the game exe before patching to strip Valve's SteamStub DRM (original kept as .steamstub.bak). No-op when there's no stub.
        • Steamless experimental
          Realigns sections during unpack for better compatibility with heavily-protected EXEs. Only matters when auto-unpack is on.
        • Auto-apply community fix (CrakFiles)
          On every fix, automatically downloads and applies a confident-match community fix from the CrakFiles list (only an exact name match with a pixeldrain link; overwritten files are backed up). On by default — disable it if you don't want AW downloading/overwriting game files from a community source.
        • Steam API ownership-check bypass
          Off by default. Drops SteamAutoCrack's bypass proxy (winmm.dll) + a redirect config so games that re-check the original steam_api/exe after the DLL swap still launch and unlock. Downloads the proxy from its official release.
        • Create Launch.bat for fallback
          Unused — ColdClient was removed.
        • Check for GBE Fork updates
          Re-checks GitHub for the latest GBE Fork build before applying (downloads only if newer). Disable to always use the cached build.
        • GBE/Goldberg — download icons
          When enabled, the auto-repair that writes a missing steam_settings/achievements.json also downloads achievement icons from Steam CDN.
        Setup & maintenance
        • First-run guide
          Reopen the setup guide for API key, profile, folders, sources, and notifications.
        • Fix all detected games
          Run the emulator fix (GBE Fork, achievements, DLCs) on every detected emulated game that has a known install folder.
        • Disable hardware acceleration
          Turn off GPU acceleration if the overlay or window misbehaves. Requires a restart.

          Platform guide

          What Achievement Watcher scans, which folders matter, and when generated emulator configs are useful.

          First steps
          • Add save/config folders in Settings > Folders, then enable the matching source in Settings > Sources.
          • Use library folders for installed games that have not produced achievement files yet.
          • Watchdog only notifies new changes after the folder is known and the source is enabled.
          Steam emulators
          • GBE / Goldberg saves are usually under %APPDATA%\GSE Saves or %APPDATA%\Goldberg SteamEmu Saves.
          • CODEX / RUNE style saves often live under Public Documents\Steam\CODEX or Public Documents\Steam\RUNE.
          • TENOKE, UniverseLAN, SmartSteamEmu, ALI213 and Hoodlum are detected from their config/save folders.
          Console emulators
          • RPCS3 achievements are read from dev_hdd0\home\...\trophy near the RPCS3 install.
          • shadPS4 trophies are read from its app data game_data/trophy folders; leftover trophy data is not install proof.
          • Xenia progress is read from Xbox profile files such as .gpd.
          Launchers and sources
          • Steam public profile mode lists installed or owned Steam games, but local unlocks still come from save files.
          • GOG, Epic, Ubisoft Connect/LumaPlay and EA Desktop can be enabled separately in Sources.
          • EA Desktop cracked installs are read from EADesktopVerbose.log and its backup logs.
          Generated configs
          • The right-click emulator fix can add GBE Fork files, achievements schemas, DLC data and launch helpers.
          • Simple mode is fast and uses Achievement Watcher's data; Advanced runs generate_emu_config for deeper Steam data.
          • Advanced generation can take several minutes, especially with DLC/depot metadata.
          Troubleshooting
          • If a game is missing, rescan after adding folders, then check the source toggle and blacklist.
          • If notifications are missing, check Windows Focus Assist and run the game/app with matching admin rights.
          • Advanced > Diagnostics opens logs and data folders when you need to inspect what was scanned.
